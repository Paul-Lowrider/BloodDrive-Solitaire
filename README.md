Notes
- I added `docs/.nojekyll` so Pages will serve files in `docs/` unchanged.
- I also added a GitHub Actions workflow to automatically publish `docs/` to GitHub Pages when you push to `main`. This will create the Pages site for you without needing to change repository settings manually.

If the site doesn't appear after a few minutes:
1. Check Actions → Workflows and confirm the "Deploy Pages" workflow ran successfully.
2. If the workflow is blocked, enable GitHub Actions for the repository in Settings → Actions.
